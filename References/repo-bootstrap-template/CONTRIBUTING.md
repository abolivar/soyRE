# Contributing

## Flujo de trabajo

1. Leer `AGENTS.md`.
2. Leer la documentación del módulo.
3. Trabajar en tickets pequeños.
4. Evitar cambios fuera de alcance.
5. Ejecutar validaciones.
6. Crear PR pequeño y revisable.

## Validación mínima

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm build
```

## Pull requests

Cada PR debe incluir:

- Resumen de cambios.
- Ticket relacionado.
- Archivos modificados.
- Cómo se probó.
- Riesgos o pendientes.
